Что нужно для запуска:
Установи Python(при установке поставь галочку Add Python to PATH): https://www.python.org/downloads/windows/
Установи Flask(выполни в cmd после установки Python): pip install flask
Скачай Elib2Ebook(Elib2Ebook-windows-latest-portable.zip): https://github.com/OnlyFart/Elib2Ebook/releases
Распокуй архив в папку \\vpn_site\work
Открой app.py при помощи текстого редактора и измени путь к рабочей папки. По умлочанию стоит "E:\vpn_site\work". Обязательная для работы структура папки work:
(папка)Patterns, Elib2EbookCli.exe, (папка)old
Запусти run.bat. Если всё прошло успешно, в консоле должно появиться: "Running on http://127.0.0.1:8080". Указанный мной адрес доступен исключительно с устройства, где запущен хост.
Так же, в консоле появятся все остольные адреса, по которым достуен UI, если вы находитесь в локальных сетях.
Устройства из общей с хостом локальной сети смогут получить доступ к сайту(в том числе, из сети vpn) по соотвествующему сети адресу.

Как скачивать:
Не закрывая консоль, открой сайт в браузере. Интерфейс будет выглядеть примерно вот так(тёмная тема в комплекте не идёт. Возможно, пока):
<img width="1048" height="463" alt="image" src="https://github.com/user-attachments/assets/37a36c5a-6a2d-4165-8b45-adc1d80b434b" />

В поле "Ссылка", нужно ввести ссылку на то, что вы хотите скачать. Например: https://ficbook.net/readfic/10878444
В поле "Пароль" нужно ввести пароль, который можно задать в файле app.py, изменив значение в ковычках: PASSWORD = "MEOW". По умолчанию установлен пароль "MEOW".
Пароль нужен для того чтобы те, кто находятся с вами в одной локальной сети не забили ваш компьютер кучей скаченых книг и не использовали ваше устройство для DDOS атак. Это нужно, если вы используйте публичный vpn.

После введения ссылки и пароля, начнется скачивание, лог будет выводится ниже(нумирация глав в порядке и качаются все, просто в данном примере автор решил сделать так):

Назначения кнопок и функции:
После скачивания, появятся 3 кнопки:
<img width="1010" height="169" alt="image" src="https://github.com/user-attachments/assets/f9c3f286-c057-4f97-87de-d8017a0bd124" />

Первая передает скаченный файл на устройство, где открыт сайт, вторая перемещает файл в мамку "old". Пока доступ к ней можно получить на устройстве хоста. Третья удаляет скаченный файл.
Так же, если какой-то скаченный файл уже присудсвует в папке, при скачивании нового, старый автомачески будет перемещён в архив.
